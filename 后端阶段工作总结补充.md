# 《一次买够》网购比价平台后端阶段工作总结补充（可落地操作版）

## 一、文档定位
本补充报告不再做纯讲解，聚焦**下一阶段可以直接落地在项目中的操作清单**，按“任务 → 具体操作 → 影响文件/目录 → 产出 → 验收方式”形式输出。

---

## 二、下一阶段工作内容总览（P0/P1）

### P0（必须优先落地）

#### 1) 数据持久化改造（替换内存数据）
- **具体操作**
  1. 新增数据库连接与配置层（建议 `app/db.py`），读取环境变量。
  2. 新增 ORM 模型与迁移（建议 `app/models.py` + `migrations/`）。
  3. 将 `app/data.py` 由内存数据改为数据访问层（CRUD/查询/分页）。
  4. 更新接口实现：`app/main.py` 中调用持久层。
- **影响文件/目录**
  - `app/data.py`、`app/main.py`、`app/schemas.py`（必要时字段调整）
  - 新增：`app/db.py`、`app/models.py`、`migrations/`
  - `requirements.txt`（补充 ORM/驱动依赖）
- **产出**
  - 可运行的数据库 schema + 迁移脚本
  - 主要业务数据可持久化（商品、收藏、提醒、历史、搜索记录）
- **验收方式**
  - 本地启动后数据重启不丢失
  - CRUD 接口返回与 `openapi.json` 一致

#### 2) 认证升级（演示 Token → 生产级 JWT/会话）
- **具体操作**
  1. 更新 `app/auth.py`，接入 JWT 生成/校验/刷新逻辑。
  2. 增加 token 过期、刷新、黑名单策略。
  3. 在 `app/main.py` 的鉴权依赖中统一处理 401/403。
- **影响文件/目录**
  - `app/auth.py`、`app/main.py`、`app/schemas.py`
  - `requirements.txt`（JWT 库依赖）
- **产出**
  - 标准 JWT 登录与刷新机制
  - 统一错误码与过期处理逻辑
- **验收方式**
  - 登录/刷新/登出流程可复现
  - 过期 token 访问得到稳定 401 响应

#### 3) 统一错误码与参数校验治理
- **具体操作**
  1. 新增错误码字典（建议 `app/errors.py`）。
  2. 在各接口统一抛出业务异常并映射为 `ApiResponse`。
  3. 补齐关键参数校验（价格区间、分页值、平台编码等）。
- **影响文件/目录**
  - `app/main.py`、`app/schemas.py`
  - 新增：`app/errors.py`
- **产出**
  - 错误码标准化清单
  - 统一异常响应结构
- **验收方式**
  - 传入错误参数可得到一致错误码
  - 前端无需分支解析

---

### P1（高优先，可并行推进）

#### 4) 降价提醒异步化（任务化落地）
- **具体操作**
  1. 新增任务模块（建议 `app/tasks.py`）。
  2. 接入定时任务/消息队列执行降价扫描。
  3. 写入触发记录与通知状态。
- **影响文件/目录**
  - `app/tasks.py`、`app/data.py`、`app/main.py`
- **产出**
  - 可运行的提醒扫描任务
  - 触发结果可查询
- **验收方式**
  - 目标价触发后记录更新
  - 任务日志可追踪

#### 5) 真实数据接入（采集/聚合）
- **具体操作**
  1. 新增数据源接入层（建议 `app/providers/`）。
  2. 建立采集/同步策略（定时/增量）。
  3. 对接数据清洗与去重逻辑。
- **影响文件/目录**
  - 新增：`app/providers/`
  - `app/data.py`（聚合入口）
- **产出**
  - 可接入至少一个真实平台数据源
- **验收方式**
  - 搜索接口返回真实商品数据

#### 6) 联调与测试补强
- **具体操作**
  1. 扩展 `tests/test_api.py` 覆盖收藏、提醒、历史等主流程。
  2. 增加接口契约校验（对齐 `openapi.json`）。
  3. 明确联调脚本/请求样例（Postman/HTTPie）。
- **影响文件/目录**
  - `tests/test_api.py`
  - `openapi.json`
- **产出**
  - 覆盖主要业务链路的测试用例
- **验收方式**
  - `python -m pytest -q` 全量通过

## 三、可直接在项目补充的操作清单（执行顺序建议）

1. **新增数据库配置与模型层**（`app/db.py` + `app/models.py`）
2. **替换 `data.py` 为持久层实现**（查询、分页、排序）
3. **认证升级 + 错误码治理**（`auth.py` + `errors.py`）
4. **提醒任务化落地**（`tasks.py` + 定时任务）
5. **真实数据接入与同步策略**（`providers/`）
6. **完善测试与联调脚本**（`tests/` + 文档样例）
